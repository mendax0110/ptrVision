Im\+Gui\+File\+Dialog is a file selection dialog built for (and using only) \href{https://github.com/ocornut/imgui}{\texttt{ Dear Im\+Gui}}.

My primary goal was to have a custom pane with widgets according to file extension. This was not possible using other solutions.\hypertarget{md__2_users_2adriangossl_2_documents_2git_2ptr_vision_2external_2_im_gui_file_dialog_2_documentation_autotoc_md222}{}\doxysection{\texorpdfstring{Features}{Features}}\label{md__2_users_2adriangossl_2_documents_2git_2ptr_vision_2external_2_im_gui_file_dialog_2_documentation_autotoc_md222}

\begin{DoxyItemize}
\item C Api (succesfully tested with Cim\+Gui)
\item Separate system for call and display
\begin{DoxyItemize}
\item Can have many function calls with different parameters for one display function, for example
\end{DoxyItemize}
\item Can create a custom pane with any widgets via function binding
\begin{DoxyItemize}
\item This pane can block the validation of the dialog
\item Can also display different things according to current filter and User\+Datas
\end{DoxyItemize}
\item Advanced file style for file/dir/link coloring / icons / font
\begin{DoxyItemize}
\item predefined form or user custom form by lambda function (the lambda mode is not available for the C API)
\end{DoxyItemize}
\item Multi-\/selection (ctrl/shift + click) \+:
\begin{DoxyItemize}
\item 0 =\texorpdfstring{$>$}{>} Infinite
\item 1 =\texorpdfstring{$>$}{>} One file (default)
\item n =\texorpdfstring{$>$}{>} n files
\end{DoxyItemize}
\item Compatible with Mac\+Os, Linux, Windows, Emscripten, Android
\item Supports modal or standard dialog types
\item Select files or directories
\item Filter groups and custom filter names
\item can ignore filter Case for file searching
\item Keyboard navigation (arrows, backspace, enter)
\item Exploring by entering characters (case insensitive)
\item Custom places (bookmarks, system devices, whatever you want)
\item Directory manual entry (right click on any path element)
