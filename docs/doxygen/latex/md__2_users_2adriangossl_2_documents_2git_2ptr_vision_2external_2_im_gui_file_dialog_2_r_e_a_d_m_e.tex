Im\+Gui\+File\+Dialog is a file selection dialog built for (and using only) \href{https://github.com/ocornut/imgui}{\texttt{ Dear Im\+Gui}}.

My primary goal was to have a custom pane with widgets according to file extension. This library is designed to be dropped into your source code rather than compiled separately.

From your project directory\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{mkdir\ lib\ \ \ \ <or\ 3rdparty,\ or\ externals,\ etc.>}
\DoxyCodeLine{cd\ lib}
\DoxyCodeLine{git\ clone\ https://github.com/aiekick/ImGuiFileDialog.git}
\DoxyCodeLine{git\ checkout\ master}

\end{DoxyCode}


These commands create a {\ttfamily lib} directory where you can store any third-\/party dependencies used in your project, downloads the Im\+Gui\+File\+Dialog git repository and checks out the Lib\+\_\+\+Only branch where the actual library code is located.

Add {\ttfamily lib/\+Im\+Gui\+File\+Dialog/\+Im\+Gui\+File\+Dialog.\+cpp} to your build system and include {\ttfamily lib/\+Im\+Gui\+File\+Dialog/\+Im\+Gui\+File\+Dialog.\+h} in your source code. Im\+Gui\+File\+Lib will compile with and be included directly in your executable file.

If, for example, your project uses cmake, look for a line like {\ttfamily add\+\_\+executable(my\+\_\+project\+\_\+name main.\+cpp)} and change it to {\ttfamily add\+\_\+executable(my\+\_\+project\+\_\+name lib/\+Im\+Gui\+File\+Dialog/\+Im\+Gui\+File\+Dialog.\+cpp main.\+cpp)}. This tells the compiler where to find the source code declared in {\ttfamily \doxylink{_im_gui_file_dialog_8h_source}{Im\+Gui\+File\+Dialog.\+h}} which you included in your own source code.\hypertarget{md__2_users_2adriangossl_2_documents_2git_2ptr_vision_2external_2_im_gui_file_dialog_2_r_e_a_d_m_e_autotoc_md242}{}\doxysection{\texorpdfstring{Requirements\+:}{Requirements:}}\label{md__2_users_2adriangossl_2_documents_2git_2ptr_vision_2external_2_im_gui_file_dialog_2_r_e_a_d_m_e_autotoc_md242}
You must also, of course, have added \href{https://github.com/ocornut/imgui}{\texttt{ Dear Im\+Gui}} to your project for this to work at all.

Imgui\+File\+Dialog is agnostic about the filesystem api you can use. It provides a IFile\+System you can override for your needs.

By default you can use dirent or std\+::filesystem. you have also a demo of uisng boos filesystem api in the Demo\+App branch

\begin{DoxyItemize}
\item C Api (succesfully tested with Cim\+Gui)
\item Separate system for call and display
\begin{DoxyItemize}
\item Can have many function calls with different parameters for one display function, for example
\end{DoxyItemize}
\item Can create a custom pane with any widgets via function binding
\begin{DoxyItemize}
\item This pane can block the validation of the dialog
\item Can also display different things according to current filter and User\+Datas
\end{DoxyItemize}
\item Advanced file style for file/dir/link coloring / icons / font
\begin{DoxyItemize}
\item predefined form or user custom form by lambda function (the lambda mode is not available for the C API)
\end{DoxyItemize}
\item Multi-\/selection (ctrl/shift + click) \+:
\begin{DoxyItemize}
\item 0 =\texorpdfstring{$>$}{>} Infinite
\item 1 =\texorpdfstring{$>$}{>} One file (default)
\item n =\texorpdfstring{$>$}{>} n files

\item Compatible with Mac\+Os, Linux, Windows, Emscripten, Android
\item Supports modal or standard dialog types
\item Select files or directories
\item Filter groups and custom filter names
\item can ignore filter Case for file searching
\item Keyboard navigation (arrows, backspace, enter)
\item Exploring by entering characters (case insensitive)
\item Custom places (bookmarks, system devices, whatever you want)
